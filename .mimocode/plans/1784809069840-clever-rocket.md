# Curriculum Detail Page - Implementation Plan

## Overview

Build a dedicated **Curriculum Detail** page at `/secretary/curricula/:id` that provides a complete view and management interface for a single curriculum, with courses organized hierarchically by Year Level → Semester.

---

## Scope

- **Role**: Secretary (with VPAA/program_head read access via existing RBAC)
- **Route**: `/secretary/curricula/:id` (new `:id` param route — first in the codebase)
- **Backend**: All endpoints already exist from previous work — no backend changes needed
- **No modal-in-modal**: Inline expandable panel for adding courses

---

## Backend (No Changes Needed)

All required API endpoints already exist:

| Method | Endpoint | Purpose |
|--------|----------|---------|
| `GET` | `/curricula/{id}/full` | Returns curriculum + courses grouped by year_level/semester with unit totals |
| `POST` | `/curricula/{id}/courses` | Attach course (`{ course_id, year_level, semester }`) |
| `DELETE` | `/curricula/{id}/courses/{course_id}` | Remove course from curriculum |
| `PATCH` | `/curricula/{id}/status` | Activate/deactivate/archive |

```
│ │ ...                                       │   │
│ └───────────────────────────────────────────┘   │
│ YEAR 2 accordion                                │
│ ...                                             │
└─────────────────────────────────────────────────┘
```

**State**:
- `curriculum` — single curriculum metadata
- `terms` — grouped courses from API
- `courses` — all available courses (for combobox)
- `isAddPanelOpen` — inline add panel visibility
- `addCourseSearch` — search text for course combobox
- `addCourseId`, `addYearLevel`, `addSemester` — form fields
- `yearFilter`, `semesterFilter` — toolbar filters
- `collapsedYears` — accordion state per year level
- `isActivating` — loading state for activate button

**Key behaviors**:
- Fetch curriculum via `GET /curricula/{id}/full` on mount
- Fetch all courses via `GET /courses` for the combobox
- Toolbar filters filter the displayed terms (client-side)
- Accordion toggle per year level (collapse/expand)
- Activate button only visible when `status === 'draft'`
- Add panel stays open after each successful addition
- Remove course calls `DELETE /curricula/{id}/courses/{course_id}` then refetches

## Component Design Details

### Header Section
- Back arrow link: `← Back to Curricula` → navigates to `/secretary/curricula`
- Code badge: `bg-[#C9952A]/10 text-[#C9952A]` (existing brand style)
- Name: `text-2xl font-bold font-display`
- Status badge: green (active), gray (draft), red (archived)
- Metadata row: Department, Program, Version, Academic Year as subtle text
- **Activate button**: Only rendered when `status === 'draft'`. Style: `bg-emerald-600 text-white hover:bg-emerald-700`. Calls `PATCH /curricula/{id}/status` with `{ status: 'active' }`, then refetches.

### Toolbar Section
- **+ Add Course** button: `bg-[#4e0a10] text-white` — toggles inline panel
- **Year Level filter**: Native `<select>` with All/1st-4th Year options
- **Semester filter**: Native `<select>` with All/1st/2nd/Summer options
- **Search**: `<Search>` icon input, filters course code/name within displayed terms

### Inline Add Course Panel
- Renders directly below toolbar (not a modal)
- **Course combobox**: Custom searchable dropdown (following TopBar.tsx pattern):
  - Button showing selected course or placeholder
  - Dropdown with `<SearchField>` + scrollable listbox
  - Filtered by search text matching course_code or course_name
  - Shows course code + name + units in each option
- **Year Level**: `<select>` dropdown (1st-4th Year)
- **Semester**: `<select>` dropdown (1st/2nd/Summer)
- **Add** button: Calls `POST /curricula/{id}/courses`, then refetches data, clears selection, panel stays open
- **Cancel** button: Closes panel

### Year Level Sections (Accordion)
- Each year level is a collapsible section
- Header: `Year 1` with ChevronDown/ChevronRight toggle + course count badge
- Uses `collapsedYears` state (`Record<number, boolean>`)
- Animation: `transition-transform duration-200` on chevron rotation

### Semester Tables (within each year)
- Each semester gets its own table inside the year section
- Table header: semester label (e.g., "1st Semester") + total units summary
- Columns: Course Code, Course Title, Lecture Units, Lab Units, Total Units, Actions
- **Total summary row**: Bold row at bottom showing column totals
- **Remove action**: Red trash icon per row → calls `DELETE /curricula/{id}/courses/{course_id}` with confirmation

### Empty States
- No courses at all: "No courses in this curriculum yet. Click '+ Add Course' to get started."
- No courses matching filters: "No courses match your filters."
